Five protocol families in; one normalized stream out across 20+ MQTT topics. The digital twin and ECHO subscribe to all of it, while systems I don't own — the citizen, school and hospital portals — plug in over their own APIs. The pipeline's whole purpose is that single destination: one live operational picture of the city.

The Challenge

Vendor-siloed dashboards, no unified view, no path to a digital twin. Handed a problem with no existing architecture and no engineering team.

The Build

Solo-designed a Go ingestion service that normalizes multi-protocol, multi-vendor payloads into clean JSON on AWS — serving structured data to the full downstream stack.

The Outcome

Every vendor dashboard replaced. Digital twin enabled. New vendors and zones actively onboarding; alerting, analytics and a 3D city model in active development.

How a single reading travels — edge to operator

01 · INGEST
Arrive
A frame lands — a Modbus register read, a LoRaWAN uplink, a vendor SDK callback.
02 · DECODE
Unpack
The right per-protocol decoder reads the raw bytes by their documented layout.
03 · VALIDATE
Sanity-check
Readings are checked against physical reality; impossible values are dropped, not stored.
04 · NORMALIZE
Reshape
Mapped to one shape — type / id / channel — with real units, whatever the source.
05 · PUBLISH
Stream
Onto its MQTT topic, where the digital twin and ECHO pick it up in real time.

Under the Hood

Built to not fail

A city control room can't go dark. The engineering that isn't on screen is the part that keeps the data flowing at 3 a.m.

Onboarding

Config, not code

A new device is a config entry and a container restart — never a code change or a redeploy. One binary runs device #1 and device #1,000.

Isolation

Failure stays local

Each device family runs in its own isolated Docker stack, so a fault in one protocol can never take down the others.

Recovery

Self-healing

Health-gated startup ordering, MQTT auto-reconnect and restart policies mean a 3 a.m. reboot recovers with zero touch.

Security

Minimum exposure

Secrets kept out of source, cloud access via IAM roles, and security groups that expose only what genuinely must be reachable.

Network

Private mesh

WireGuard links every site into one private network; field segments never face the public internet.

Design

Lean by intent

The acquisition layer holds no state — it decodes, validates, normalizes and forwards, and nothing more. The boring choice is the reliable one.
